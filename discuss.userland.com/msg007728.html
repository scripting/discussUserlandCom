The Bean Scripting Framework (which apparently has little to do with beans) looks like a light version of OSA or WSH for Java. There's no embedding or recording, but it provides a generic way to call from Java code into scripts. The interpreters are pluggable. It also provides a way to call back into Java code from scripts, but most scripting engines that are written in Java already provide that.

I see that Josh is already thinking about BSF and Frontier...
