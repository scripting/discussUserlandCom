It sounds like you are wanting a GPL that allows a developer to sell software, be it shrink warp, subscriptin, etc. As it stands now this effect can be achieved by negiotating with each author of the code one wishes to use. Why not make this an explicit option? Create a open source liscense that requires compensation as a factor of sale price and amount of code used or importance of code to the final product. Would this fill in the gap between the for sale developers and the traditional open source developers?
