Postscript. It worked for no one besides me and Brent.

We were both using alphas of Pike, they had some stuff that got cleaned out before it went beta.

We got to the bottom of the problem and have a fix.

Back in a couple of minutes.

Step 1

Download and install:

http://static.userland.com/weblogsCom/gems/discussUserLand/pikeResponder.fttb

(There was a change to the GET script, it now calls pike.init to make sure everything is properly set up. My copy of Pike already had run this. Yours quite possibly hadn't.)

Step 2

Quit Pike if it's running.

Open pikeStartupCommands.txt in Notepad or SimpleText.

Add this line at the end of the file:

user.webserver.responders.default.condition = false;

Relaunch Pike.

Try the Firewalls with Piking Sauce experiment once again.
