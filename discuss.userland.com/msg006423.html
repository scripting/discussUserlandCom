I've created two test files:

http://www.innovision.com/xmlprotocols/sn1.xml http://www.innovision.com/xmlprotocols/sn2.xml

The first assumes that a restricted set of HTML tags (not escaped) are allowed within text elements and that HTML entities are converted to the numeric equivalent (for now I've removed the DOCTYPE declaration, so validating parsers won't choke on the HTML markup).

The second simply wraps the content of text elements within CDATA sections.

Does anyone out there have IE5 and enough knowledge of their style sheet implementation to try to render these documents and report any differences in the way the text elements look? Maybe someone knows how IE (or in general) browsers might/should treat HTML markup embedded in CDATA sections?
