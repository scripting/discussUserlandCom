What if this were taken to the next level, so that a site editor could offer a set (sub-set?) of available themes, which any user could select from? Wouldn't that be an interesting level of interface personalisation?

While Themes were in development, one thought was that a Managing Editor should be able to preview a Theme on his or her own site, without actually changing the site's Theme. But it meant adding a level of abstraction in a bunch of places. These same changes would also be needed to allow user-selectable themes.
