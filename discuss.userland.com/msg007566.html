Any progress on the blank page problem?

Author:erik
Posted:6/18/1999; 4:16:27 PM
Topic:Any progress on the blank page problem?
Msg #:7566
Prev/Next:7565 / 7567

Awhile back several people noted how Frontier can serve up a blank page (the background color and graphics header are the only thing that visibly loads, after watching the downloaded KB count increment almost all the way up to 100% Examining the source code, it appears that most of the page was downloaded, but it abruptly ends in the midst of some HTML. The error in logbrowser's inetd section is port = 81/stream = 1/whatWentWrong = Can't write stream because TCP/IP error code 10035 - Resource temporarily unavailable./whatWereWeDoing = Returning data). This happens from Netscape to Userland.com as well as my own F6-based dynamic websites. It appears to be a function of the overall time in loading the page. More than 30-40 seconds and bammo! you're out! Any progress on a fix? It's a bummer when you want to show off your Frontier based app. and you get a blank page.
