A new version of file.reconcileFolder.

Download here. It now takes a errorFilter callback, an optional parameter, and if there's an error it's called, and if it returns true, the processing continues. This allows the caller to log errors, but allows folders to still be backed up even if there are a few difficult files.

Should we release this? I think so.
