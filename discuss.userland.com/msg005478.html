FWIW, I tried renaming some directories.

Directories above Frontier (Frontier 6.0) seem like you can rename as long as you do a global Search and Replace.

Directories that are peers of Frontier... maybe not a good idea. Like Guest Databases. The name Guest Database seems to be hardcoded in the mainResponder. Once I renamed Guest Databases, when Frontier starts up it creates a new Guest Databases directory and populates it.
