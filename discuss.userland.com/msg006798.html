I've seen this happen many times with Userland web pages, let's say on average once a day, and occasionally on my own Frontier-served web pages. Unless it is a coincidence, that might suggest ruling out the ISP. The page background gets painted, maybe the header, and then it eventually stops trying to load anything else on the page (this in Netscape v4.x). This morning I had the problem and the browser was on the same machine as Frontier's web served dynamic page, it persisted when I pressed the "Reload" button, and I fixed it by quitting out of Frontier, and restarting. Next time I'll try quitting out of the browser and see whether that does anything. About the time this morning that I saw the problem on my Frontier web server, there was an inetd error in the error log:

client = 16.x.yyy.zz port = 81 stream = 25 whatWentWrong = Can't write stream because TCP/IP error code 10035 - Resource temporarily unavailable. whatWereWeDoing = Returning data

I really would like to see the problem identified, as it makes Frontier look unreliable.
