XUL allows you to build the entire UI from soup to nuts with progressbars, tabs, springy elements, etc. Dreamweaver's menus.xml file allows you to configure the tree hierarchy of the top level menu - the pulldowns.

So, XUL is more comprehensive.

But adding new UI components to Dreamweaver is very easy, so I imagine there will be "front ends" in both applications.
