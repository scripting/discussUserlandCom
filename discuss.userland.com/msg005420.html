I've made some adjustments to the Cocoon servlet as a way of testing to see if dynamic content could be used. The fruits of my labor can be found here... http://208.204.52.53:10000/test.xml?userName=josh

What happens is that the servlet checks to see if the physical file exists, if it doesn't, the servlet makes an xml-rpc request to an xml-rpc server(thanks Hannes!!). This request currently retrieves this file, http://www.stonecottage.com/josh/blurbs.xml and then the servlet uses this xsl file, http://208.204.52.53:10000/xsl/headlines.xsl for the formatting.

Though the info isn't coming from a database, it could very easily be modified to do so.

This is pretty cool and serving xml from Frontier and formatting it with xsl would be really nice and powerful.
