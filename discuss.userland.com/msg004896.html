On Macs, both QuarkXPress and Frontier support Apple events. So QuarkXPress is scriptable on Macs.

Apple events don't exist on Windows. The rough equivalent is COM. In Frontier for Windows, one uses VBScript, JScript, or PerlScript to do COM scripting. But is QuarkXPress for Windows scriptable via COM? I don't know, and a quick search of the website didn't mention it.

Frontier for Windows won't be able to script QuarkXPress for Windows unless QuarkXPress is somehow scriptable. I'd suggest checking the QuarkXPress docs for any mention of scripting, COM, automation, or VBScript.
