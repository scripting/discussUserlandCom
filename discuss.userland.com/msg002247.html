In response to the question:

I have kept the root updated using the Update Frontier command in the Main menu. Do I still need to run the updater?

Yes. To upgrade you must not just replace the app, but also run the upgrade script.

What does the upgrader do that the "Update Frontier" command doesn't? (beside providing the new app)

It tests the version of the app that you're running to ensure it is updated.

It adds some objects that are not a part of the root updates process. These are usually kernelized verbs that require a new version of the application.

On Windows, it makes sure the COM DLL is the latest version.
