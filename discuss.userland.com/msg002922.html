There's another way, but I don't think it will work with the older '$' URL scheme. In my WebSiphon applications, I include a search argument with a random value:

http://www.foo.com/script.t?rnd=5738294

Since this value is virtually always different, the browser will reload the page from the server. The script, of course, ignores the parameter.
