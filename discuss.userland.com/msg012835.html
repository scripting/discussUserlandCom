one problem that I ran into in my NetGravity days was that double slash "//" is not valid in a URL. If I remember correctly, rfc 2396 specifies that "/" is a reserved character and may be handled specially, and if you want to use any of these reserved characters, they need to be escaped.

hope this helps. -paul
