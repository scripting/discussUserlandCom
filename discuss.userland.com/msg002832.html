This would be ideal! My only addition would be a standard way to also query a server to discover the appropriate URL, MIME type, and other "dialect-specific" bits of info specific to a particular RPC implementation.

If RPCs were self-documenting, it really eliminates a lot of the need/burden related to operating some RPC registry. Just being able to ask for a text page that describes a RPC would be sufficient in many cases. Sort of like the way Unix commands without any arguments usually cough up some simple help.
