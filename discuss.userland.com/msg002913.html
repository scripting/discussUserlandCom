Found the data!!!

Here's the access to the body of an MTTF message:

responseXML.documentElement.childNodes.item(0).childNodes.item(0).childNodes.item(0).childNodes.item(0).childNodes.item(2).childNodes.item(1).text

We made an incorrect assumption that the response from the MTTF XML-RPC Handlers would be in the same form as http://www.mailtothefuture.com/msgReader.xml$1 (which is a small simple xml structure).

Instead we discovered through alerting the .xml properties of the item() collections that the actual response XML was formed similar to an MTTF XML-RPC Request schema.

What added to the confusion is that when looking at values at the root level the data appeared to be the same form as the msgReader XML format because the first 4 levels of nesting of the MTTF XML-RPC Request didn't contain any data themselves.
