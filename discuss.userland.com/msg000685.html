Interesting side effect when working in XML: these weird character entities are not defined. Dave, you might want to *not* convert the characters to HTML entities when rendering XML, instead replacing them with the nearest eqiuvalent.

The only entities defined by default in XML are <, >, &, ', and ". A workaround is, of course, to declare a DTD that includes the other entities ;)
