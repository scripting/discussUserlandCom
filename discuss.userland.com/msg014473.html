Pet peeve time. Jakob Nielsen has given us some incomplete advice. He says that readers rarely venture beyond the bottom of a page. I think Jakob is talking about pages on e-commerce sites that sell sneakers or record albums. When it comes to writing docs or specs, please don't break them up. It means more navigation for the reader and renders the Find command in the browser useless. Here's an example of a successful spec that's all on one page. It wouldn't have worked as well if it were split across several pages.
