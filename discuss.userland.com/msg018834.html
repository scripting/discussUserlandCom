Does anyone know of an XML DTD for web project management?

ie: - defining and tracking budget, deliverables, responsibilities, timelines

- localisation issues (tracking which pages/sections have been translated or localised)

- defining inheritance of attributes within a site section

- enabling exchange of data among project management, content creation (Dreamweaver) and content management (Frontier/Manila)

- setting markers and checkboxes to track what pages/sections have been completed and reviewed. Etc etc

My questions are:

Will Macromedia's Whirlwind address these issues?

Has it been done by anyone else?

Is it worth doing?

If noone's done it is there any interest in starting work on a standard?
