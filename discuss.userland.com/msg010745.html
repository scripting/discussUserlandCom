It makes one wonder... CORBA does have a big gnarly set of specifications and is not easy to use... but there is a Dynamic Invocation Interface buried in there. It allows one to dynamically create the set of stuff required for a remote method invocation and send it off to a broker for dispatch. One good use of the DII (YAA*) is interfacing to/from scripting languages, and yet, it sits around mostly unused. Perhaps some Python/CORBA jocks could explain why?
