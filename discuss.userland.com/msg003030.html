Turning off the MTTF ad wrappers does nothing to enable HTML mail.

For an HTML mail reader to interpret HTML in the message the content-type mail header parameter must be set to "text/html" or "multipart/alternative".

MTTF sets the content-type to "text/plain"

text/html Is the easy way as it's a single part message. The downside is an HTML mail client will render the tags, if the message is received by a non-html mail client then... you'll see the tags in the message.

multipart/alternative is more involved because there has to be two parts to the message; a text only version of mail message and an html version of the mail message. This allows non-html mail clients to show the text only version of the message, and html mail clients will render the html version.
