My take over the world idea last week was a rich browser interface to a web mail site, along the lines of the specialized browser for Ebay.

In a nutshell, combine storage and functionality server side with a rich lightweight interface client side. Make the client configuration a url. One file on the client, nothing to maintain.

Since all your mail is on the server, multiple interfaces for location independant access to the same data are not that hard. (basic http, xmlrpc, imap, whatever)
