The indexName parameter is not available through the mainResponder search engine XML-RPC interface. Which means that the only way to use the indexName parameter is via custom scripts.

The reason for this is you want to be able to ensure that public search engine submissions don't go to the wrong place.

A better solution to this may be devised in the future, so you can still have the control but get the extra flexibility of specifying indexName for your sites.

In the meantime, you'll have to write custom scripts if you want to index to gdbs other than Index.root.
