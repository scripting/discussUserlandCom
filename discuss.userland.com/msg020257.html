Code exhibitionism?

Author:David Adams
Posted:8/25/2000; 1:34:40 PM
Topic:A softer GPL?
Msg #:20257 (In response to 20255)
Prev/Next:20256 / 20258

How would you describe a software developer (as opposed to someone or an organization that doesn't write code) as part of an "open source" or "truly free" software community who, for the most part, doesn't release OS or TF software?

Here's a proposal: If they release some, but not all or even most of their code, maybe we should term them "Code Exhibitionists" or "Code Teases," whereas people who always release their code could be called "Code Nudists" or "Code Naturists." :-)
