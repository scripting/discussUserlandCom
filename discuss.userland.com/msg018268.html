Hmmm, what makes you think this is about *you* Mr Developer?

Perhaps MS are doing this for their own internal development purposes. They've long said they like programming in Java. There are things they want to change before they write WindowsXYZ in Java. This is what C# is for.

They can move their C/C++ corpus to C# reasonably easily. An important thing here is that the generated code fits in with COM. Looking beyond the immediate syntax and runtime of the language. The code tree could be compiled to bytescodes, or to native code. One signal thing for me is decorating the source tree with all these little attributes.

Look at some other movements - Charles Simonyis' pet - Intentional Programming - that group has moved into the Visual Tools group. 'The IP Project is moving from Research into Development, to gear up for larger scale internal use and eventual shipping.'

Quite a few of the Functional Programming resarchers around are now at Microsoft Research, they are avid tree transformers, decorators, code-compilers. Simon Peyton-Jones. Look at his paper 'Aspect Oriented Compilers'. There's a tip of the hat to the IP group in there.

Predications - the current Win32 API will dissapear into a COM like thing. Everything in the OS will become a COM like object. C# will be used for system level components in Visual Tools environments. Next Intensional Programming will be launched in the VisualStudio enviroment. You will edit higher levels abstractions 'Intensions' and these will be transformed to lower level code. Visio will be pulled in as a way to edit using diagrams - to draw software.

Look at filter-graphs in DirectShow.

Anders Heijslberg on C# (formerly architect of Delphi). What MS are doing with these people and Bill and Charles is constantly reappraising and developing the way they write component software. This may mean your software, but it definately means theirs.
