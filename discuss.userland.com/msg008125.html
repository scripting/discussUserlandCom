Think of the c++ program as a website (no macros, no process directives, no template). Render to an ftp-filewriter, which ftps to the linux box. That reduces the hard part of version control and getting the files to the linux box to a known and solved problem.

As a post render, make a http request to apache/zope/python/whatever xml-rpc for a compile/make. I.e, the request needs to be path to makeile and the make target. Make compiles and returns the results.
