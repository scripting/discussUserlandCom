X already supports networked displays. It's not exactly what you say: programs run on the remote machine and display on the local machine. The drawback to this is that you need a very fast connection to be usable. The advantage is that you don't need to have the software installed on the local machine to use it.

This can be used both to run individual programs remotely, or to get your whole session from another machine (using the X Display Manager Control Protocol).
