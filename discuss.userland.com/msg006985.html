Are you running console.root and news.root on the same machine?

For some reason, you are probably getting an HTML error message returned from the server instead of a proper XML-RPC response.

To debug this problem, you could temporarily add "fldebug:true" to the calls to betty.rpc.client in consoleSite.glue.newStoryPost and consoleSite.glue.editStoryPost (on the client machine).

Then post or edit a story. Two windows will open displaying the XML-RPC request and the result obtained from the server. What does the XML-RPC response look like?
