  Is there a convenient way to 'upload' a guest database to a remote Frontier server from a local desktop? - We have a setting where we might want to put guest databases for our customers onto a remote server.

The only way I could imagine now would be to create a new database remotely and then upload the individual tables via WebEdit. Is this the correct way?

That's how I do it, but I just use it to get stuff from the NT to the Mac so I can save it on a ZIP drive for transport between facilities.

You could write an XML-RPC handler that creates a GDB on the server (make sure you secure it). Then you could invoke the RPC to create a new GDB with the right name, and use WebEdit to send over the GDB contents, one top-level table at a time. And finally, use WebEdit to make a new entry in the server Frontier's user.databases table, so the GDB is reopened when Frontier or the machine is restarted.

Or, you could fall back on a non-Frontier solution: If your server has an FTP server, you could save the GDB and copy it to the server in the correct location. Then you'd need to be able to tell Frontier (on the server) to open the new file.
