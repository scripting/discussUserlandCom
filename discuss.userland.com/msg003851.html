I agree with the previous poster who didn't like how a site's content only moved to a permanent location after it moved off the main page, so there was no reliable address for a current piece of news. (MacOS Rumors was the particular site that bugged me)

My Solution

I render the current content in its permanent place at the same time I render it on the 'dynamic' page.

At the bottom of my 'front' page, http://nowthis.com/log/, I put the following (which is automatically generated each time I post):

This page's content changes every few days. This particular entry's permanent home is: http://nowthis.com/log/1999/03/09.html

(with the URL changing each time, obviously)

It's not hard to do.
