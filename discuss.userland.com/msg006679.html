Following up on my own thread with another idea for a UI improvement.

The first one should be very simple to implement and rather useful if you subscribe to/follow WEBLOGS that don't get updated EVERY day. Right now the top line of each sites description box contains a TIME stamp indicating when that page was last updated. I'd like to add one more thing.

• If the date of the pages last update is NOT equal to the CURRENT date please add the actual update date to the status line.

This will let people catch sites whose update process has failed, or maybe skip over sites they have already read, and/or sites that haven't been updated yet TODAY.

• Another possible option(in the time control theme) would be to only display sites that were updated today, or maybe since your last check of the my.userland.com site. This would reduce the amount of material that would have to be skipped over by the reader(its already been read). It could also reduce the load on the server by reducing the number of sites that have to be formatted(stop at date check). The reduced page size/content would also cut down on bandwidth requirements for the server or connection.

==Paul
