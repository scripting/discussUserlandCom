Came across a problem with running Frontier 5.1.6 behind IIS using the ISAPI filters: If you have checked the Basic Authentication option in the Properties -> Security tab for your default website in MMC, authentication headers don't get through to Frontier (as far as I can tell).

The solution is to uncheck it for the whole website (in MMC) and use it only on those sub-sections that require it.
