I believe the upcoming RMI-IIOP has been precisely designed to take care of that.

Java RMI-IIOP beta (You'll need to registery w/ JDC to access this)

Here's the blurb:

RMI-IIOP provides the Java developer an implementation of Java Software's JavaTM RMI API over the Object Management Group's industry-standard Internet Inter-Orb Protocol (IIOP). It is a standard extension to the Java Development Kit (JDKTM). RMI-IIOP is based on the two new OMG standards:

  • The Java-to-IDL mapping
  • Objects-by-Value

With it, developers can write remote Java interfaces and implement them just using Java and the Java RMI APIs. These interfaces can be implemented in any other language that is supported by an OMG mapping and a vendor supplied ORB for that language. Similarly, clients can be written in other languages using IDL derived from the remote Java interfaces. Using RMI-IIOP, objects can be passed both by reference and by value over IIOP.
