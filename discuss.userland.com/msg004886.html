No, I don't mean discuss.root, actually. If you're using the sample discussion group, your discussion group website is in the discussionGroup table in htmlInterfaces.root. That discussionGroup table needs a #search table.

Of course, you may have put your discussion group website somewhere else -- but the upshot is that it, like any other searchable website, needs a #search table.

http://frontier.userland.com/tree$3.2.7

Furthermore, the database your discussion group website is in should have a table in user.databases, and supportsIndexing should be set to true in that table.

http://frontier.userland.com/tree$3.2.4.6
