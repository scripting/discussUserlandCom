I found using root-relative references (ie dropping 'localhost' and just starting with a slash '/') works fine in the #url tables.

This will stop working the minute you start to reference content hosted on multiple (virtual) servers from the #urls table. You will obviously have to include the server name at that point. As long as you are aware of that, I don't see a problem with this approach.
