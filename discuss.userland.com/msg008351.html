sure would be great... although it is possible to use the current version of FM for XML. I am currently use FMPro 4 and a few other doodads (including FileMakerLib, xmltr and some other frontier scripts) to take records out of FM into XML tables, then into an XML outline, through xmltr and into web pages. It's a circuitous route, but it works. Search the DG for more info on xmltr - it's a very nice Frontier suite for processing XML.
