That's right, it shouldn't be. There are several good models out there to start with -- Scripting News, More Like This, and Wilma, just to name a few. And with some of the tools that are already available, you can even translate between DTDs fairly easily.

The point is this: Yes, you will have to write Robot Wisdom differently if you want to use XML. You will *not* have to write it twice, however. You can write it once in XML and use your choice of stylesheet languages to render it in HTML, RSS, LaTeX, plain text, etc.
