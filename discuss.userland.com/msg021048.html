Unless [the GPL] mentions something about runtime binding

Unfortunately, it doesn't say enough. This vagueness, combined with the Cover Your Ass principle, means that most people make conservative assumptions. The consensus that I see is that if any code loaded into a process is GPL'ed, all of the code must be GPL-compatible (ignoring any explicit GPL exceptions). So DLLs don't matter.
