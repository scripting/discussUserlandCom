Main responder pushes it out with Content-type + "text/html". Is there any way of forcing/persuading main responder to change this to "application/pdf" so that the browser knows what to do?

Yes, mainResponder looks at the user.webserver.prefs.type2MIME table to convert odb types to MIME types, mainResponder.getODBMimeType is the script that does the actual work.

In fact, user.webserver.prefs.type2MIME already contains an entry for the binary type 'PDF '. When you look at the PDF file you imported, what binary type are they?
