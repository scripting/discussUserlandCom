If you've got a linux box handy, nmap is a really useful network scanning tool. It's availiable from http://www.insecure.org .

With a couple of commands, you can scan your machine for any open ports, both tcp and udp.

It's not the answer, but it's a way to find it.

eric

Later... This only catches server type sockets, not client initiated connections. I believe that the way to check the client connections (fron nt) is some variation of the netstat -a command. Of course if you _really_ want to know what is happening, tcpdump is your friend.
