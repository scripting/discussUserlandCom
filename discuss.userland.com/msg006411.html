Since it isn't parsed, well-formedness+validity can't be checked...

If the purpose of the tag is for plain text, wellformedness/validity should not be a concern. Plain text is just a string of text, it's up to the "receiver" to do any parsing and/or validation. Or, the text string should be validated before it went into the CDATA. In another word, within the scope of the "scripting news format", is just a string of text, thus tags inside should be treated as plain text as well.
