None of this stuff requires JAVA.....

A lot of people have proposed to emulate RMI in a non-Java environment. I tried to explain yesterday why I think this is not a good idea. Emulating RMI means to emulate the whole Java class library. Your system would practically have to think in Java classes. What I could see is having a thin Java layer to mediate between Jini and the non-Java system.
