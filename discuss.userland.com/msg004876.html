Some script is looking for system.temp.mainResponder.newsRootFile. This entry is created on startup by mainResponder.startup if a file named news.root exists in the www sub-folder of your Guest Databases folder.

So your problem should be solved by creating such a file (your news site needs to live in it), quitting and relauching Frontier.

An updated sample news.root is scheduled to be released shortly.
