1. Error-proofed the server, if there's an error, such as a missing message, the result returned will be in XML, not HTML. The result, instead of containing a , will contain an , which is a string, the text of the error message produced by Frontier.

2. If any of the text contains any <'s we replace them with &lt;s. Some early messages contained HTML markup, somewhere along the line we disallowed this.
