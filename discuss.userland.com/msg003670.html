There can be no summary of GUI programming with Linux mainly because there is no One Way to do it! Each GUI library has their own documentation (or lack thereof). The Qt library (which has the advantage of being Unix/MS-Windows cross-platform) is pretty well documented. There are also several Qt books coming out Real Soon Now.

In general, any programming on Linux (GUI or not) is like any programming on any Unix variant.
