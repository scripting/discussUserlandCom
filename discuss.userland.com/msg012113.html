Do you mean 'the same language' or 'the same byte codes'? There are other languages that execute as Java bytecodes - NetRexx, Ada, and Python, to name a few.

Regardless, the point is well-taken, XML-RPC and SOAP will not be supporting the migration of code from one place to another.

Regardless, SOAP could serve as an RMI transport, although it might not support all of the features of RMI (pass-by-value).

I'm not suggesting that someone do this - just wanted to make sure that everyone has the full story.
