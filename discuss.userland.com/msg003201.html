I fixed a couple of minor bugs in the superCalendar() macro that affected optional parameters passed to the macro:

1) When passing a value to cellHeight to specify the number of rows per table cell, the macro was actually generating one fewer rows than requested. Confusing. I fixed that, which may require you to retune that parameter.

2) If you pass a list of addresses to adrCalendar in order to merge multiple event lists onto one calendar, you are supposed to be able to pass either a RGB string specifying one color for all entries, or a list of strings with as many elements as the list of adrCalendar addresses. The single-string option caused the macro to fail. Now, it works. (Thanks to Steve Poole for tripping on this one.)

The updated version, 1.01, has replaced the original version in the fat page at 

    http://www-perg.phast.umass.edu/IDB/frontier/calendarScript/
If anybody else has found bugs or deficiencies, I'd like to hear about them.
