Well, what about an xml model for themes then? Suck in an xml model for the theme from an external source and any locally defined element overrides the external one. Define it by standard elements: home page, news box, discuss page, etc.

It's not a very elegant inheritance mechanism, but then it's probably a hell of lot less work. Fine granularity for inheritance can be a remarkably tricky thing to implement...

To be honest, I don't see subscribable themes really going far and wide for interest (a cool concept, no question) but I can see it helping large organizations really get their house in order without massively destructive centralization having to take place.
