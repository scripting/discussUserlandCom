I'd like a search-engine-friendly URL to accompany each story. That URL should point to the content *only* of the story (possibly in plain text with no newlines to prevent people from using that link to avoid banners); no navigation tools, no banners, no extras.

This is the key to solving the scalability problem of the search engines. If each site had an RSS that pointed to everything that was new on the server, in a search-friendly format; and if each search engine had an email address rss-incoming@alltheweb.com; and if every site mailed their RSS whenever it changed - then everything in the search engines would be up-to-the-minute and coverage would be 100% (of the RSS-aware servers).

There would be trust problems, of course, but the search engines would be free to index only RSS files from trusted sites, or to compare one URL to the other.

/Sten
