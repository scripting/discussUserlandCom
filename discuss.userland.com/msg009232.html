The problem with "just write it and then get it adopted" is that once servers are widely deployed it becomes very, very difficult to change them. (It's actually easier to get vendors to support a completely new protocol on their clients & servers than it is to make sure everyone is using the same version of a protocol that's undergoing revision.)

Why does it matter? Because you absolutely MUST be able to change the protocol as necessary to pass IESG security review, and to address scalability issues.

Participate in the IETF process. Don't try to subvert it.
