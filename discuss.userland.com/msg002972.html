Today I went live with a modified version of imageRef, which reformats the imageUrl into an absolute path pointing to port 81 which is handled by IIS. Frontier serves all text pages via mainResponder, all images are handled through IIS.

Check out http://www.egger-lerch.at/ The image on the frontpage is linked through a random script.

The patch is very easy: Two lines in html.getImageData and a new #pref in the website table which specifies the port. The images are still stored in the GDB, they are written out when needed. No change in the way how to build a website with Frontier.

PLEASE NOTE: Work on a copy of imageRef and html.getImageData because these scripts are Userland area.

I tried this because my NT setup went unstable using the ISAPI plug-in for Frontier. (I had a strange behaviour of IIS)
